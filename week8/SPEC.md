# Week 8 – 笔记应用需求文档

## 一、项目概述

- **项目名称**：NoteHub
- **类型**：多栈笔记管理应用
- **核心功能**：用户可以创建、阅读、编辑、删除笔记，并支持分类组织和全文搜索
- **目标用户**：个人用户，用于日常笔记记录与整理

---

## 二、功能需求

### 2.1 核心功能（CRUD）

| 操作 | 说明 |
|------|------|
| **Create** | 创建新笔记，包含标题、内容、所属分类 |
| **Read** | 查看单条笔记详情；查看所有笔记列表 |
| **Update** | 编辑笔记的标题、内容、分类 |
| **Delete** | 删除笔记 |

### 2.2 分类功能

- 用户可以创建、编辑、删除分类（文件夹）
- 每条笔记归属一个分类（可选，不强制）
- 支持在分类视图中筛选该分类下的所有笔记
- 默认提供一个"未分类"分类

### 2.3 搜索功能

- 支持按**标题**和**内容**关键词搜索
- 搜索结果实时返回，支持高亮匹配词

---

## 三、数据模型

### 3.1 笔记（Note）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | integer | 主键，自增 |
| title | string | 笔记标题，必填，最大 200 字符 |
| content | text | 笔记内容，可为空 |
| category_id | integer | 所属分类 ID，可为空 |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

### 3.2 分类（Category）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | integer | 主键，自增 |
| name | string | 分类名称，必填，唯一，最大 100 字符 |
| created_at | datetime | 创建时间 |

---

## 四、非功能需求

- 每个技术栈版本均需包含持久化存储（数据库或文件）
- 基本输入验证（标题非空、分类存在等）
- 错误提示（友好错误页面或提示信息）
- 提供 README.md 说明本地运行方式
- 三个版本功能保持一致，便于横向对比

---

## 五、技术栈要求

| 版本 | 技术栈 | 说明 |
| ------ | -------- | ------ |
| 版本 1 | 需使用 `bolt.new` 生成 | 首选 AI 生成方式 |
| 版本 2 | 需使用非 JS 语言 | 例如 Django + Python |
| 版本 3 | 自选（MERN / MEVN / Next.js 等） | 独立于版本 1 和 2 |
