# 第六周 — 使用 Semgrep 扫描并修复漏洞

## 作业概述

使用 **Semgrep** 对 `week6/` 目录下的应用进行静态分析。筛选发现的问题并修复至少 3 个安全问题。在您的报告中，解释 Semgrep 发现了哪些问题以及您如何修复它们。

## 了解 Semgrep

Semgrep 是一个开源的静态分析工具，可以搜索代码、发现错误并实施安全防护和编码标准。

1. 点击[此处](https://github.com/semgrep/semgrep/blob/develop/README.md)了解 Semgrep。

2. 按照上述链接中的安装说明进行操作。您可以选择使用 **Semgrep Appsec Platform** 或 **CLI 工具**。

## 扫描任务

### 扫描目标

- 后端 Python (FastAPI)：`week6/backend/`
- 前端 JavaScript：`week6/frontend/`
- 依赖项：`week6/requirements.txt`
- 配置/环境文件（用于查找密钥）：`week6/` 目录下的文件

### 运行通用安全扫描以及针对密钥和依赖项的专项扫描

从**作业仓库根目录**运行以下命令，应用包含代码和密钥规则的 CI 风格规则包：

```bash
semgrep ci --subdir week6
```

## 任务

1. 选择 Semgrep 发现的任意 3 个问题，并使用您选择的 AI 编码工具进行修复。

2. 展示精确的修改内容并解释缓解措施（例如：参数化 SQL、更安全的 API、更强的加密、净化 DOM 写入、限制 CORS、依赖升级）。

3. 重要：确保修复后应用仍能正常运行，且测试仍然通过。

## 交付物

### 1. 发现概述

- 总结 Semgrep 报告的类别（SAST/密钥/SCA）。
- 标注您选择忽略的误报或干扰规则及其原因。

### 2. 三个修复（修复前 → 修复后）

对于每个修复的问题：

- 文件和行号
- Semgrep 标记的规则/类别
- 简要风险描述
- 您的更改（简短的代码差异或说明，使用的 AI 编码工具）
- 为什么这能缓解问题

## 提示

- 优先选择针对根本原因的最小化修改。
- 每次修复后重新运行 Semgrep，确认问题已解决且没有引入新问题。
- 对于依赖项，记录升级版本并在使用了供应链扫描时附上相关安全公告链接。
