# Task 1 PR 描述：添加更多端点和验证

## 问题描述

Week 7 的 Task 1 要求为后端 API 添加更多端点，并实现输入验证和错误处理。当前后端仅有基础的 CRUD 端点，缺少删除端点和输入验证逻辑。

## 实现方案

1. **新增删除端点**
   - `DELETE /notes/{note_id}` — 删除指定笔记
   - `DELETE /action-items/{item_id}` — 删除指定待办事项

2. **添加输入验证**
   - 使用 Pydantic schema 对请求体进行验证
   - 对必填字段、字符串长度等进行限制

3. **完善错误处理**
   - 404 返回明确信息
   - 400 返回验证错误详情

## 权衡与限制

- 删除端点未实现级联删除（如有标签关联需先清理）
- 未添加软删除机制，历史数据无法恢复
- 后续可考虑添加批量删除功能
