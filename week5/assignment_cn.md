## 探索起始应用

一个极简的全栈起始应用。

- FastAPI 后端 + SQLite（SQLAlchemy）
- 静态前端（无需 Node 工具链）
- 极简测试（pytest）
- Pre-commit（black + ruff）
- 用于练习代理驱动工作流程的任务

以这个应用为实验场，尝试你使用 Warp 构建的自动化工具。

### 项目结构

```text
backend/                # FastAPI 应用
frontend/               # 由 FastAPI 服务的静态 UI
data/                   # SQLite 数据库 + 种子数据
docs/                   # 代理驱动工作流程的 TASKS
```

## 第一部分：构建你的自动化工具（选择 2 个或更多）

从 `week5/docs/TASKS.md` 中选择任务来实现。你的实现必须通过以下两种方式使用 Warp（详见下方）：

- A) 使用 Warp Drive 功能 — 如保存的提示词、规则或 MCP 服务器。
- B) 在 Warp 中整合多代理工作流程。

保持你的更改集中在 `week5/` 内的后端、前端、逻辑或测试上。
对于每个选中的任务，请注意其难度级别。

### A) Warp Drive 保存的提示词、规则、MCP 服务器（必需：至少一个）

创建一个或多个适用于此仓库的可共享 Warp Drive 提示词、规则或 MCP 服务器集成。示例：

- 带覆盖率和失败测试重跑的测试运行器
- 文档同步：从 `/openapi.json` 生成/更新 `docs/API.md`，列出路由变更
- 重构工具：重命名模块、更新导入、运行 lint/测试
- 发布助手：版本递增、执行检查、准备变更日志片段
- 集成 Git MCP 服务器，让 Warp 自主与 Git 交互（创建分支、提交、PR 备注等）

>*提示：保持工作流程专注于单一任务，传递参数，使其具有幂等性，并尽可能选择无头/非交互式步骤。*

### B) Warp 中的多代理工作流程（必需：至少一个）

运行一个多代理会话，在不同的 Warp 标签页中让独立的代理并发处理任务。

- 使用并发代理在不同的 Warp 标签页中执行 `TASKS.md` 中的多个自包含任务。挑战：你能让多少个代理同时工作？

>*提示：[git worktree](https://git-scm.com/docs/git-worktree) 在这里可能有用，可以防止代理之间相互覆盖。*
